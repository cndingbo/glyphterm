# 安全策略

## 支持的版本

| 版本 | 支持 |
|------|------|
| `main` 分支最新提交 | ✅ 积极修复 |
| 已发布标签（若有） | ✅ 仅严重漏洞 |
| 更旧版本 | ❌ 不保证 |

## 报告漏洞

**请勿在公开 Issue 中讨论可利用的安全问题。**

请通过以下方式私下报告：

1. GitHub **Private vulnerability reporting**（若仓库已开启），或
2. 向仓库维护者发送邮件（请在 [GitHub 个人主页](https://github.com/cndingbo) 查看联系方式）

请尽量包含：

- 影响组件（如 SSH 会话、`glyphvt` 解析、Tauri 命令面）
- 复现步骤与 PoC（如有）
- 影响评估（RCE、凭据泄露、DoS 等）

我们会在 **72 小时内** 确认收到，并尽力在 **14 天内** 给出修复计划或缓解措施。

## 范围说明

**在范围内：**

- GlyphTerm 自带的 PTY/SSH 会话处理、VT 解析、IPC/Tauri 命令
- 依赖的已知 CVE 若影响默认构建路径

**通常不在范围内：**

- 用户 shell 配置（`.zshrc`、插件）导致的命令执行
- 远程服务器上的恶意软件或社会工程
- 仅影响开发依赖、未进入发行版的工具链问题

## 安全设计原则

- **本地优先**：凭据与历史默认留在本机
- **最小权限**：SSH 使用用户提供的密钥或 agent，不擅自提升权限
- **输入边界**：PTY/SSH 字节流经 VT 状态机解析，过滤已知有害 OSC（如 shell integration 序列）
