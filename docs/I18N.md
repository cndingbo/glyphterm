# UI internationalization

GlyphTerm desktop UI defaults to **English**. Users can switch to **简体中文** from the title bar **Language** control.

## Locales

| ID | Language |
|----|----------|
| `en` | English (default) |
| `zh-Hans` | Simplified Chinese |

Preference is stored in `localStorage` under `glyphterm-locale`.

## Adding strings

1. Add the key to `apps/glyphterm-desktop/src/i18n/locales/en.ts`
2. Mirror the same tree in `zh-Hans.ts`
3. Use `t("section.key")` in TypeScript or `data-i18n="section.key"` in HTML

Interpolation: `t("ssh.failed", { error: "..." })` → `{error}` in the message.

## Static HTML

Elements with `data-i18n`, `data-i18n-title`, `data-i18n-aria`, or `data-i18n-option` are refreshed on locale change via `applyI18nToDocument()`.

Dynamic regions (welcome panel, terminal idle, activity rail) subscribe to `onLocaleChange()`.

## Theme names

Theme picker uses `themeDisplayName()` — English `name`, Chinese `nameZh` from presets.
